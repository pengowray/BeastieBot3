    Ways to get IUCN data — what each unlocks

    Each row is a download/build option. ✓ = it adds that piece; · = it doesn't; the last column lists extra requirements. The API options are additive — typically cache-taxa → infraranks → cache-assessments → project-view, or iucn api cache-all --full in one job.

    Option Species Subspecies & varieties Orphan subspecies † Removed / historical Rich content ‡ Ready to generate lists Requires
    iucn import (CSV release) ✓✓✓ ·~✓ manual zip download
    iucn api cache-all (taxa + assessments) ✓·· ·✓· API token; CSV (species list)
    iucn api discover-by-family +·· ✓✓· API token
    iucn api cache-infraranks (cached/API-sourced) ·✓· ·✓· API token; cached species
    iucn api cache-infraranks --from-csv ·✓✓ ·✓· API token; CSV
    iucn api cache-assessments (downloads queued payloads) ··· ·✓· API token
    iucn api project-view (build projection) ··· ··✓ — (local; needs the cache)
    iucn api cache-all --full (the chain) ✓✓✓ ·✓✓ API token; CSV

    + discover-by-family adds species the CSV omits (removed/historical/reclassified). † Orphan subspecies = an assessed subspecies whose parent species is unassessed; it appears on no family page and in no cached species, so only the CSV's sis_id reaches it. ‡ Rich content = synonyms, population narratives, trends and history the CSV doesn't carry. "Ready to generate lists" means you can run generate-lists/charts afterwards (CSV via the default --dataset csv; the API cache only after project-view via --dataset api).
